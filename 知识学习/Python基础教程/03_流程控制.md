# 流程控制

## 程序的流程控制

- 顺序结构：代码自上而下，逐行执行
- 分支结构：根据条件，分情况执行
- 循环结构：根据条件，重复执行

## 导入模块

- Python 使用 import 导入非内置的模块。
- 常用的写法：
  - 将某个模块整个导入：import 模块 。导入后需要使用"模块名.函数名()"调用函数。
  - 将某个模块整个导入：import 模块 as 别名。导入后需要使用"模块别名.函数名()"调用函数。
  - 将某个模块的所有函数导入：from 模块 import *。导入后直接使用"函数名()"调用函数。
  - 将某个模块的某个函数导入：from 模块 import 函数。导入后直接使用"函数名()"调用函数。
  - 将某个模块的某些函数导入：from 模块 import 函数1, 函数2, ..., 函数n。导入后直接使用"函数名()"调用函数。
- 建议使用 import ... 方法导入，使用 from ... import ... 可能会导致函数名冲突。

## 生成随机数

- 导入 random 库。`import random`
- `random.randint(a,b)` 生成一个大于等于a且小于等于b的随机整数
- `random.randrange(a,b)` 生成一个大于等于a且小于b的随机整数
- `random.random()` 生成一个大于等于0且小于1之间的随机浮点数
- `random.uniform(a,b)` 生成一个a和b之间（包含a和b）的随机浮点数

### 例子：判断某年是否为闰年

- 现行为主流历法为格里高利历，其置闰规则为：
  - 一般情况（非世纪年），如果年份数除以4可以除得尽，则置闰，即2月有29天，全年有366天。
  - 特殊情况（世纪年：即最后两位为00的年份），如果年份数除以400可以除得尽，则置闰，即2月有29天，全年有366天。
  - 其它情况，为平年，不置闰，即2月有28天，全年有365天
- 为什么要置闰？
  - 经天文测量，目前一回归年约为365.2422天。
  - 如果不置闰年，则一年有365天，误差为0.2422天/年。
  - 如果不处理世纪年，则一年有365.25天，误差为0.0078天/年。
  - 如果处理世纪年，则一年有365.2425天，误差为0.0003天/年。

```python
# LeapYear.py
year = eval(input("Enter a year: "))

# Check if the year is a leap year 
# method 1
isLeapYear = False
if year % 100 == 0:
    if year % 400 == 0:
        isLeapYear = True
else: 
	if year % 4 == 0:
        isLeapYear = True
        
# method 2
isLeapYear = (year % 4 == 0 and year % 100 != 0) or (year % 400 == 0)

# Display the result 
print(year, "is a leap year?", isLeapYear)

```

## 循环结构

### while语句

- 遇到while语句时，如果条件成立，while执行循环体中的语句
- 执行完每一轮循环后，while会重新判断条件，如果条件仍然成立，则再次执行循环体中的语句

```python
cond1 = True
while cond1:
    pass
```

### 例子：从1累加到100求和

```python
i=1
sum=0
while i<=100:
	sum += i
	i += 1
print(sum)
```

- 在执行到while语句时，条件如果为False，则直接跳过循环，执行循环之后的语句。
- 在循环体中，必须有使条件判断结果发生变化的语句，否则循环将永远执行下去（称为死循环）。

```python
x = eval(input())
while x!= 0:
    print(x)
    x -= 3
print("Finish.")
```

### for...in语句

- for...in语句一般用于实现已经循环次数的循环。
- 基本结构：for 变量 in 序列，循环执行的次数与序列中元素的个数相同，每次循环中，变量自动赋值为序列中的某个元素。

### range()函数

- range()函数经常用在for...in语句中

- range(n)：产生一个 0,1,2,3,....,n-1 这n个数的序列，注意是从0开始，总数为n个。

  - 如果 n<= 0，生成的序列为空。

  - 例：range(5)：产生 0,1,2,3,4 这 5 个数组成的序列

- range(a,b)：产生从 a 到 b 的数字序列，注意是大于等于a且小于b，即包含 a 而不包含 b ！

  - 如果b<=a，生成的序列为空。
  - 例：range(1,10)：产生 1,2,3,...,9 这 9 个数的序列

- range(a,b,k)函数：产生一个a,a+k,a+k+k,...序列，即从 a 开始向 b 行进，但不能达到 b，即：

  - 如果 k>0，生成序列中的最大值不能大于或等于b。如果b<=a，生成的序列为空。
  - 如果 k<0，生成序列中的最大值不能小于或等于b。如果b>=a，生成的序列为空。
  - k不能等于0。省略参数 k，相当于 k=1 
  - 例：range(0,10,2)，将产生0,2,4,6,8这个序列
  - 例：range(10,1,-1)，将产生10,9,8,7,6,5,4,3,2这个序列

### break 和 continue

- break语句可以跳出 for 和 while 的循环体
- 例：循环输入一个数字，输出其平方，直到输入0为止。

```python
while True:
    x = int(input())
    if x == 0:
        break
    print(x**2)
```

- continue语句跳出当前轮循环，直接进行下一轮循环
- 例：打印100以内的正整数中，不可以被7整除，数字中也不含7的整数。

```python
for i in range(1,101):
    if i % 7 == 0:
        continue
    if i % 10 == 7:
        continue
    if i // 10 == 7:
        continue
    print(i)
```

- 此语法为Python特有，其它语言（如C语言）一般没有将else用于循环语句中的语法。

```python
    print(number,"is a prime number")

    print(number,"is not a prime number")

```

- 循环中使用else，如果不执行break，则执行else

```python
number = int(input())
for i in range(2,number):
    if number % i == 0:
        break
else:
    print(number,"is a prime number")
```

- 优化：其实循环结束的上限，不必达到 number-1，而是达到 math.ceil(number**0.5) 即可。原因：如果 n 是合数，则 n 必存在一个因数a小于等于n的平方根。证明如下：
  - 设 m = math.ceil(n\*\*0.5)，如果存在 a,b 且 a<=b，使得 a\*b = n，那么必然 a<=m。
  - 反证法：假设a>m。因为a<=b，所以b>m，所以  a\*b > m\*\*2 ，已知 n=a\*b，代入上式，n > m\*\*2，与m的定义矛盾。

```python
number = int(input())
isPrime = True
for i in range(2,math.ceil(number**0.5)+1):
    if number % i == 0:
        isPrime = False
        break
if isPrime:
    print(number,"is a prime number")
else:
    print(number,"is not a prime number")
```



## 异常处理

### 使用try语句及except子句

- try用于将可能引发异常的代码隔开
- except用于标记当异常出现时需要执行的代码

```python

# try/except
try:

	print("Something went wrong!")
```

## 调试程序

### 断言（assert）

- 断言：assert 条件(True/False), “条件为False时显示的提示信息”。
- 假如你预期此处x的值为0.3，如果不是0.3，则抛出异常。

```python
>>> x=0.1+0.2
>>> assert x==0.3, "Ooop, x is wrong"
Traceback (most recent call last):
  File "<stdin>", line 1, in <module>
AssertionError: Ooop, x is wrong
```

- 断言针对的是程序员的错误，而不是用户的错误。
- 对于可以恢复的错误，请抛出异常，而不是用 assert 语句检测它。
- 运行python时使用 -O 选项，可以禁用断言。

### 日志（logging）

- 使用日志 logging 记录程序的运行过程和状态
- 不要用print()调试。会与正常的输出混淆。
- 日志是给程序员看的，不是给用户看的。

### 日志级别和日志函数

| 日志级别 | 日志函数           | 描述                                             |
| -------- | ------------------ | ------------------------------------------------ |
| DEBUG    | logging.debug()    | 调试，最低级别，调试诊断时的问题                 |
| INFO     | logging.info()     | 信息，一般事件的信息，或确认一切正常             |
| WARNING  | logging.warning()  | 警告，表示可能存在问题，不会阻止程序的工作       |
| ERROR    | logging.error()    | 错误，会导致程序失败                             |
| CRITICAL | logging.critical() | 严重，最高级别，表示致命错误，会导致程序完全停止 |

```python
import logging
logging.basicConfig(level=logging.DEBUG, format='%(asctime)s - %(levelname)s - %(message)s')
x = 1
logging.debug("x="+str(x))
# 2023-03-09 15:28:14,204 - DEBUG - x=1
```

- 禁用日志： logging.disable(日志级别)，等于或低于此级别的日志禁用
- 禁用所有日志：logging.disable(logging.CRITICAL)
